• 서버와 클라이언트(Server and Client)

    앵귤러 어플리케이션은 Http 혹은 Websocket을 이용하여 서버와 데이터를 주고 받을 수 있다. 서버에 요청을 보내려면 Http Object를 사용할 수 있으며, Websocket또한 Observable (Chapter4) 과 맵핑할 수 있다.

  • Http Object

    앵귤러 어플리케이션은 Http Request을 비동기(asynchronous)로 처리하기에 Http Request을 보낸 후에도 사용자가 하던 작업을 계속할 수 있다(No refresh). Asynchronous Http Request는 Observable (Chapter4)로 처리할 수 있으며, Observable이 프로미스보다 훌륭한 선택임은 당연하다.

    Http 클래스는 @angular/http 패키지에 정의되어 있다. Http 객체의 함수들은 Http Protocol Method를 정의하고 있으며, RequestOptionArgs 객체를 추가하여 옵션(Header,Parameter)등을 설정할 수 있다. Request객체는 Http 객체보다 더 많은 옵션을 제공하며, Header를 정의하고, 요청 방식을 선택하는 등 Http 요청 방식을 더 세밀하게 조정할 수 있다.

    Request 객체를 생성하면서 객체 리터럴을 전달하지 않고, RequestOptionsArgs 객체를 전달할 수도 있다.
