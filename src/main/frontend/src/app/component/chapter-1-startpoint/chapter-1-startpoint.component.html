  • Component 기본

    컴포넌트(Component)는 Angular 어플리케이션을 구성하는 기본 요소이며, 템플릿을 가진다

    모든 앱은 최소한 하나의 모듈(Root Module)과 하나의 루트 컴포넌트(root component)을 가진다.

    클래스의 @Component 선언시, 해당 클래스는 컴포넌트로 선언된다

    컴포넌트 어노테이션안의 메타데이타(Metadata) 종류에는 selector, template 등이 있으며 기능은 아래와같다.


    • selector : 컴포넌트에 고유의 HTML 태그를 부여한다
    • template 혹은 templateUrl : 템플릿(view) 의 정보를 명시한다
    • styles 혹은 styleUrls: 스타일 스크립트 의 정보를 명시한다

    컴포넌트가 위치할 selector 와 template은 최소 요구 사항이며, 뷰와 컴포넌트 로직을 별개로 설계하여 관심이 다른것을 분리(seperation of different concerns) 것이 컴포넌트의 핵심이다.

    {{escapeHtml1}} 은 컴포넌트의 프러피티인 name을 템플릿으로 렌더링함을 의미한다

  • Module 기본

    모듈(Module>은 컴포넌트(Component) , 서비스(Service), 디렉티브(Directive)을 논리적인 하나의 요소로 모아놓은 것이다.

    모듈은 @NgModule 어노테이션을 붙여 선언하며 어노테이션안의 Metadata에 모듈의 내용을 설정한다. RootModule (bootstrap될) 에서는 반드시 BrowserModule을 불러와야 하며, 그외 모듈은 CommonModule을 불러와 사용한다.

  • 데이터 바인딩(Data Binding) 기본

    데이터 바인딩(Data Binding)이란 컴포넌트의 프로퍼티(property)와 뷰(View or Template)을 동기화시키는 작업을 의미한다.

    데이터 바인딩에는 세가지 종류가 있으며 모두 단방향이다

    • {{escapeHtml4}} : 삽입식(injection)
    • {{escapeHtml5}} : 프러퍼터 바인딩(property or attribute binding)
    • {{escapeHtml6}} : 이벤트 바인딩(event binding)

    Hello {{name}}!

    The field is not valid

    양방햔 바인딩(bi-directional data binding) [(ngModel)]로 구현 가능하지만 꼭 필요하지 않다면 사용하지 않는것이 바람직하다. 또한 템플릿 안에서 다른 엘리먼트의 프러퍼티를 지역 템플릿 변수(local template variable)을 생성하여 참조할 수 있다.


    {{localTemplate.value}}
