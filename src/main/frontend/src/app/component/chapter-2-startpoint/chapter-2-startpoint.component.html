  • Router 기본

    앵귤러는 싱글 페이지 어플리케이션(Single Page Application - SPA)을 구축하는데 사용되며, 라우터router는 페이지 내용 일부를 변경하는데 이용된다. 어플리케이션에서는 보통 변하는 부분과 변하지 않는 부분으로 구성되는데 간략한 구성은 이러하다.

    main 은 SPA에서 변하는 부분이며, 라우터 영역 을 통하며 표시할 수 있다.


  • 위치 정책

    브라우저의 주소 표시줄은 URL(Uniform Resource Locator)을 의미한다. URL은 여러 구성 요소로 나누어지는데, 구성은 이러하다. 

            http://siwoo.com:8080/learning?myName=SiWoo
        Protocol HostName     FileName?QueryString1&QueryString2

    앵귤러에서 주소를 관리하는 위치 정책에는 HashLocationStrategy 와 PathLocationStrategy등이 있다. 기본 정책인 PathLocationStrategy에서 HashLocationStrategy를 변경시에, 루트 모듈(Root Module)에서 아래와 같이 설정한다.


  • 라우터의 구성

    라우터의 모듈은 RouterModule에 구현되어 있으며, 라우터 모듈의 구성 요소는 아래와 같다.

    라우터 구성 요소

    Router : 라우터를 구현한 객체다. 메서드로는 navigate(),navigateByUrl()등이 있다. 이 객체의 프러퍼티인 path을 변경하면 원하는 경로로 이동할 수 있다.


    RouterOutlet : 컴퍼넌트(Component)을 렌더링(Rendering)하는 영역이다 으로 지역을 지정한다

    Routes : 컴퍼넌트(Component)와 URL을 연결하는 객체의 배열(Array)이다.

    RouterLink : 앵커 태그의 Default action을 정지하고 라우터를 통하여 컴퍼넌트(Component)를 렌더링 하도록 변경한다.

    ActivatedRouter : 현재 활동하는 라우터의 인스턴스를 가리키는 객체다. 컴퍼넌트(Component)의 생성자(Constructer)로 앵귤로가 주입(Injection)한다

    라우터는 Routes(배열) 객체를 사용하여 설정한다

    라우터는 모듈 단위로 설정하며, @NgModule 어노테이션의 메타데이터에 라우터 모듈을 로드한다.

    루트 모듈이 아닌 자식 모듈(Child Module)에서는 라우터 모듈의 forChild() 메서드를 사용한다
